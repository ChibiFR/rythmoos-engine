Loader

Loader

The loader makes it easy to load and preload assets, before the game starts or on-the-fly.

Constructor

(abstract) new Loader()

Source:
  • media/Loader.js, line 28
Examples
import {Loader} from 'rythmoos-engine';

Preloading assets

import {Game, Loader} from 'rythmoos-engine';

// Create a game
const game = new Game(500, 500, document.body);

// Load assets before the game starts
game.load = () => {
  Loader.load('some file', 'assets/some-file.png');
  Loader.load('some other file', 'assets/some-other-file.mp3');
};

Loading assets on-the-fly

// Anywhere outside of the Game.load and Game.preload methods
Loader.load('an image', 'an-image.jpg').then(img => {
  // Image loaded, also available at Loader.get('an image');
});

Loader.load('a video', 'a-video.mp4').then(video => {
  // Video loaded, also available at Loader.get('a video');
});

Example
// Let's say all our assets are in the assets folder, we dont want to specify
// the assets directory in the path everytime we load an assets
Loader.basePath = 'assets';

// We can now load, for example, `assets/my-image.png` as so:
Loader.load('my image', 'my-image.png');

(static) load(name, pathname, typeopt) → {Promise.<*>}

Source:
  • media/Loader.js, line 65
Load a file.
You can tell the Loader which type your file is, or let the Loader guess it from the extension.
Example
// Load images/audios/videos or JSON
Loader.load('a file', 'a-file.jpeg').then(myFile => {
  // File is loaded, and also available from Loader.get('a file');
});

// Or load other types
Loader.load('an unknown type of file', 'unknown-type-of.file').then(myFile => {
  // File is loaded, and also available from Loader.get('an unknown type of file');
  // myFile is a Reponse object from the fetch() method
});
Parameters:
Name Type Attributes Description
name string The name to use for the file.
pathname string The path name to the file to load.
type 'audio' | 'video' | 'image' | 'json' | 'other' <optional>
 The type of file.
Returns:
A promise that should contain the loaded file.
